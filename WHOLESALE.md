# Wholesale Store — Implementation Reference

A password-protected wholesale portal built inside a single Shopify page template.
Hash-based JS navigation — no page reloads, ever. All product data embedded server-side.
No Shopify checkout. Orders submitted via a contact form; PDF export via `window.print()`.

The visual design reuses existing store CSS entirely. No new shared snippets were created —
wholesale-specific logic lives in the page template and isolated `ws-` CSS rules.

---

## Decisions

| Topic | Decision |
|---|---|
| Collections | Regular store collections, flagged with `custom.wholesale` boolean metafield |
| Wholesale pricing | `custom.wholesale_price` decimal (EUR) on each **product** |
| RRP | `custom.rrp` decimal (EUR) on each **product**, shown next to wholesale price |
| Currencies | EUR, GBP, USD — live rates from `open.er-api.com/v6/latest/EUR` on page load, hardcoded fallbacks |
| Currency selection | Buyer selects currency immediately after password, before seeing any content (once per session) |
| Password gate | Password stored in `page.metafields.custom.password`, checked against sessionStorage key `ws_access` |
| Page detection | `template.suffix == 'wholesale'` everywhere — never based on URL or page handle |
| Data loading | All data embedded server-side as `wsData` JS object in the template (no AJAX, no Storefront API) |
| Collections filter | Truthy check: `{% if collection.metafields.custom.wholesale %}` (not `== true`) |
| Product card price | Product-level wholesale price, displayed via `data-ws-eur` attribute, refreshed on currency change |
| Product detail layout | Reuses `.product-page` / `.product-main` / `.product-info` CSS exactly — same as regular product page |
| Product detail right panel order | Title → colour name → colour swatches → description → currency switcher → wholesale/RRP prices → size/qty table → Add to Order |
| Accordion sections | Removed (DETAILS / SIZE & FIT / DELIVERY & RETURNS not shown in wholesale) |
| Add to Order feedback | Button text → "Added to cart", colours inverted (white bg / black border) for 1.5s, then restores |
| Back button | Removed from product detail — browser back / menu navigation used instead |
| Cart | Full-page view inside the template (not a drawer, not a separate snippet) |
| Cart persistence | `sessionStorage` — resets on tab/browser close |
| Post-submit cart | Cart stays visible after order submission |
| Mobile | Hamburger, mobile search, mobile menu, sidebar cart all hidden on wholesale template |
| SCSS | Additive only — wholesale rules at the bottom of `scss/styles.scss` under `/* ─── Wholesale ───*/`. Compiled via Gulp. Never edit `assets/style.css.liquid` directly. |
| Wholesale JS | Lives entirely in `templates/page.wholesale.liquid`. `assets/scripts.js.liquid` is not modified. |
| jQuery timing | jQuery + Slick load after the inline template script. Auto-unlock (returning sessions) polls with `setInterval` until both are available before calling `unlock()`. |

## Architecture

### Template: `templates/page.wholesale.liquid`

Three top-level sections rendered in sequence:

1. **`wsData` script block** — server-side Liquid loop embeds all wholesale data as a JS object before any views render
2. **`#ws-gate`** — password form (hidden after auth)
3. **`#ws-currency-gate`** — currency picker (shown once per session, after password, before content)
4. **`#ws-content`** — five views, all hidden by default, shown/hidden by the JS router

### Hash routing

```
(no hash / #home)        → Home view  — wsMenuExpand(false)
#collections             → Products view, all collections merged — wsMenuExpand(true)
#collection/{handle}     → Products view, single collection — wsMenuExpand(true)
#product/{handle}        → Product detail — wsMenuExpand(true)
#cart                    → Cart + buyer form — wsMenuExpand(false)
```

`wsHandleHash()` runs on page load (inside `unlock()`) and on every `hashchange` event.
Deep links work — loading the page with a hash navigates directly after unlock.

### Entry flow

```
Page load
  └─ wsData embedded (Liquid)
  └─ Password gate IIFE runs
       ├─ Already authed (ws_access=1)?
       │    └─ Poll for $ + $.fn.slick → unlock()
       └─ Not authed → show #ws-gate form
            └─ Correct password → sessionStorage.setItem('ws_access','1') → unlock()

unlock()
  ├─ No currency chosen yet → show #ws-currency-gate
  │    └─ Currency selected → sessionStorage.setItem('ws_currency', X) → showContent()
  └─ Currency already set → showContent()

showContent()
  └─ Show #ws-content → wsUpdateCartCount() → wsHandleHash() → $(window).trigger('resize')
```

## Server-side Data Embed (`wsData`)

All data is Liquid-rendered on page load. No AJAX ever.

```javascript
wsData = {
  collections: [
    {
      handle, title, image,   // image is null if collection has no image
      products: [
        {
          handle, title,
          description,        // product.description HTML
          wholesalePrice,     // product.metafields.custom.wholesale_price (EUR)
          rrp,                // product.metafields.custom.rrp (EUR)
          colorName,          // colour name of this variant from other_colors metafield
          images: [],         // image URLs at height:1500
          variants: [{ id, title, available }],   // no prices on variants
          colors: [{ handle, color, pattern }]    // color is raw CSS value (not json-filtered)
        }
      ]
    }
  ]
}
```

**Gotchas:**
- Collection metafield filter is a truthy check, not `== true` (boolean vs string type inconsistency)
- `collection.image` and `color.pattern` are nil-guarded (`{% if ... %}...{% else %}null{% endif %}`)
- `color.color` is output as `"{{ color.color }}"` — NOT `{{ color.color | json }}`. The `json` filter serialises Shopify colour objects as `{red:…}` hashes; the bare Liquid output gives the CSS hex string.

## Header (`snippets/header.liquid`)

Detected via `template.suffix == 'wholesale'`. All changes are conditional — regular site unaffected.

| Element | Regular site | Wholesale |
|---|---|---|
| Logo href | `/` | `{{ page.url }}#home` |
| Mobile icons | Shown | Hidden (`{% unless template.suffix == 'wholesale' %}`) |
| Menu left | SHOP / ABOUT / IMAGES | COLLECTIONS toggler + `#ws-collections-list` (server-side wholesale collection links) |
| Menu right | Search + Cart + Login | `<a class="ws-cart-btn">CART (<span class="ws-cart-count">0</span>)</a>` only |
| Mobile nav + mobile search | Shown | Hidden |
| Sidebar cart drawer | Shown | Hidden |

`ws-cart-count` is a class (not ID) so `wsUpdateCartCount()` can target it via `querySelectorAll`.

`#ws-collections-list` is toggled by `wsMenuExpand(open)` — visible on collection/product views, hidden on home/cart.

## Product Cards

Built entirely in JS (`wsBuildCard`). HTML matches `snippets/product.liquid` exactly so all existing CSS applies without modification:

```html
<div class="product-card">
  <div class="product-images product-card-slider">
    <a href="#product/{handle}"><img src="..." alt="..."></a>
    <!-- one per image — slick turns into slider -->
  </div>
  <a href="#product/{handle}">
    <span>Title<br>
      <span class="regular-price" data-ws-eur="{eur}">€XX.XX</span>
    </span>
  </a>
  <div class="product-colors">
    <a class="product-color" href="#product/{color_handle}"
       style="background-color:{hex}; background-image:url({pattern});"
       tabindex="0"></a>
  </div>
</div>
```

Slick is initialised per card after DOM insertion. Must be re-run for each infinite scroll batch.

## Product Detail

Built in JS (`wsRenderProduct`). Reuses existing `.product-page` layout CSS — no new layout rules needed.

```
.product-page (flex row)
  ├─ .product-main.desktop   — all images stacked vertically (hidden on mobile)
  ├─ .product-slider.mobile  — slick slider (hidden on desktop, inited in wsRenderProduct)
  └─ .product-info.ws-product-info (sticky)
       ├─ .product-info-title > h1
       ├─ .product-color-desc  (colour name, if set)
       ├─ .product-colors      (swatches — active has no href, others link to #product/{handle})
       ├─ .product-info-desc   (product.description)
       ├─ .ws-currency-switcher
       ├─ .ws-product-prices   (wholesale price + RRP for the whole product)
       ├─ .ws-variant-table    (size / qty per row — no prices on variants)
       └─ button.ws-add-btn    (type="button")
```

`.ws-product-info` overrides `top: 125px` → `top: 10px` (SCSS, scoped to wholesale only).

The `.product-slider` slick init and accordion toggles from `scripts.js.liquid` bind statically at page load, so `wsRenderProduct` inits the slider itself after injection.

## Infinite Scroll

In-memory only — no AJAX. Data already in `wsData`.

1. `wsInitGrid(products, title)` — resets state, renders first 12, sets up IntersectionObserver
2. Observer arms only after 80px scroll (prevents immediate trigger on short pages)
3. On intersect: show loading, `setTimeout(1500ms)`, render next 12, re-arm if more remain
4. Slick inited per batch

## Cart

Order lines stored in `wsOrder` array (in-memory + sessionStorage). Cart view renders on `#cart`.

- Editable qty inputs per line
- Remove button per line
- Wholesale total + RRP total displayed side by side
- Currency switcher (same 3-button component as product detail)
- Buyer form: name, company, email, phone, address, notes
- PDF button: populates `#ws-print-view` and calls `window.print()`
- Submit: serialises order into hidden `contact[body]` field, POSTs to Shopify `/contact`

## Data Model

### Metafields

| Namespace + key | Type | On | Purpose |
|---|---|---|---|
| `custom.wholesale` | Boolean | Collection | Marks collection as wholesale-available |
| `custom.wholesale_price` | Decimal (EUR) | Product | Wholesale unit price |
| `custom.rrp` | Decimal (EUR) | Product | RRP shown alongside wholesale price |

### Currency

- Base currency: EUR
- Live rates fetched from `open.er-api.com/v6/latest/EUR` on page load
- Hardcoded fallbacks: `{ EUR: 1, USD: 1.08, GBP: 0.86 }`
- Selected currency in `sessionStorage` key `ws_currency`
- Buyer must select currency once per session (shown after password, before content)
- All price elements use `data-ws-eur` attribute; `wsRefreshPrices()` recalculates on currency change

### BE-1 — Manual per-currency pricing (fixed rounded prices)

**What:** Instead of live conversion rates applied to the EUR price, each currency should have its own manually set price.

**In practice:**
- Add new product metafields:
  - `custom.wholesale_price_gbp` (Decimal) — wholesale price in GBP
  - `custom.wholesale_price_usd` (Decimal) — wholesale price in USD
  - `custom.rrp_gbp` (Decimal) — RRP in GBP
  - `custom.rrp_usd` (Decimal) — RRP in USD
  - (EUR prices stay as `custom.wholesale_price` / `custom.rrp`)
- Embed all six values in `wsData` per product
- In `wsFormat()` / `wsRefreshPrices()`: when currency is GBP or USD, use the manual metafield value directly instead of converting from EUR. Fall back to conversion if the manual field is 0 or null
- The live exchange rate fetch can be removed or kept as the fallback only
- In the `data-ws-eur` attribute pattern: the attribute name becomes misleading. Either rename to `data-ws-base` or store all three values as `data-ws-eur`, `data-ws-gbp`, `data-ws-usd` on each price element

### BE-2 — Allow ordering out-of-stock products (showroom-level toggle)

**What:** All sizes should be orderable regardless of inventory. This is controlled at the showroom/page level, not per product, and must not affect real Shopify inventory.

**In practice:**
- Add a boolean metafield on the **Page** object: `custom.wholesale_allow_oos`
- In the `wsData` Liquid embed, read this flag: `{% assign ws_allow_oos = page.metafields.custom.wholesale_allow_oos %}`
- When building variant objects, output `available: true` unconditionally when the flag is set, rather than `{{ variant.available }}`
- In JS: the `ws-oos` class and disabled input logic in `wsRenderProduct` already reads `v.available`, so if it's forced to `true` server-side, no JS changes needed
- This does not touch `variant.inventory_quantity` or any Shopify inventory settings — it only affects what the wholesale JS sees
